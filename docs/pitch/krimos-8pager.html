Architecture
Six layers. One runtime.
Each layer does one job, composable and addressable through clean contracts.
कूप
KūpaAudit Surface
Every decision on one pane. The view operators, auditors, and regulators share.
कुल
KulaDigital Twins
A twin per role. Operators describe campaigns and workflows in plain language; the twin proposes governed strategies and instructs Karta to execute.
कर्ता
KartaAI Co-Workers
Multi-modal agents in two latency classes: sub-200ms voice, 1–30s back-office. End-to-end across customer channels and back-office workflows.
केन्द्र
KendraIntelligence Runtime
Where the thirty-three validators live. Routes models, manages context, persists memory. Pauses when a warrant fails.
क्रिया
KriyaAtomic Primitives
Five hundred-plus compliance-encoded actions. Each versioned, scoped, contract-bound. fetch_credit_bureau, validate_kyc, draft_offer…
कोविद
KovidaWorld Lending Model
The model foundation. It learns from every transaction the runtime validates, understanding lending, and your book, more precisely the longer it runs.
Composable from layer zero.
KrimOS Overview
The Runtime
Kendra.
The intelligence runtime.
The runtime is itself composed. Six sub-systems. Every action a Karta proposes passes through all of them before it reaches the real world.
Krim-Orchestra
Orchestration
Routes every request to the right model: frontier, fine-tuned, on-prem. Composes multi-step workflows with deterministic replay.
Krim-Fabric
Context & Memory
One unified context layer across structured records, documents, and conversation history. Memory persists, scoped by tenancy.
Krim-Govern
Policy Engine
Country, sectoral, and institution rules encoded once. The rules library Krim-Nyāya enforces: versioned, scoped, regulator-readable.
Krim-Nyāya
न्याय  ·  Pre-Execution Validation
Thirty-three validators from Navya-Nyāya, the 14th-century formal logic of Mithila. Pramāṇa · Doṣa · Yogyatā. Each fires before every action.
Krim-Learn
Continuous Learning
Outcomes feed back into the runtime within guardrails. The system improves, never outside its compliance envelope.
Krim-Sense
Observability
Every prompt, every validator outcome, every action streamed to an immutable audit trail. Reproducible. Exportable.
Pre-execution. Not post-audit.
