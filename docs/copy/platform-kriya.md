## 2 · Why primitives
EYEBROW: WHY IT MATTERS
H2: Small, validated, accountable.
COPY: Each primitive does one thing — with its checks built in, metered as it runs, and logged when it's done. Because the building blocks are validated, everything built from them inherits that safety. No loose ends to audit later.
PRESENTATION: one primitive shown as a clean unit — input, built-in check, output. Restrained.

## 3 · The ten categories
EYEBROW: THE LIBRARY
H2: Ten kinds of action.

## 4 · One primitive, end to end
EYEBROW: A CLOSER LOOK
H2: Watch one fire.
COPY: Take a single outbound call. Before it dials, it clears consent, calling hours, contact limits and do-not-call. It returns a transcript, an outcome, a record — and meters itself as it goes. One small action, fully governed.
PRESENTATION: the signature device — the MAKE_CALL "receipt": inputs → pre-checks (cyan→mint) → outputs → KWU. Illustrative (a lending example).
